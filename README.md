# VoidUI

> A futuristic, premium, elegant, and minimal Roblox/Luau GUI framework.

VoidUI is a complete UI library built from scratch — not a reskin of Fluent, WindUI, Rayfield, or Linoria. It takes visual cues from Arc Browser, the Unreal Engine editor, Framer, Figma, and VisionOS and folds them into one cohesive design language: dark graphite surfaces, layered floating cards, soft volumetric shadows, thin glowing borders, generous rounded corners, animated gradients, a touch of bloom, and spring-driven micro-interactions that make the whole thing feel alive.

It ships as a clean multi-file module tree for development, **and** as a single self-contained file you can drop straight into a `loadstring` call — exactly like the popular executor libraries.

---

## Why VoidUI?

- **Original look.** A distinct identity rather than "another Fluent clone."
- **Chainable API.** `VoidUI:CreateWindow(...)` → `Window:CreateTab(...)` → `Tab:CreateToggle(...)`. Reads top to bottom, writes itself.
- **Cinematic startup.** An animated splash with a glowing core, orbiting particles, and a staggered boot sequence.
- **Live theming.** Five built-in themes (Default, Midnight, Nebula, Crimson, Emerald) plus a real-time accent color that recolors the entire interface.
- **A real animation engine.** Springs, tweens, timelines, blur, glow/bloom, and particle systems — all driven by a single scheduler.
- **33 widgets.** From the basics (toggle, slider, dropdown) to the wild (code editor, graph, chart, timeline, media players, command palette).
- **Managers that do the boring work.** Config save/load with profiles, a plugin system, notifications, input/hotkeys, window z-ordering, asset caching, and layout.
- **No globals, no leaks.** Strict OOP with per-instance cleanup tracking, so `:Destroy()` actually cleans up.

## Installation

### Option A — `loadstring` (the executor way, like WindUI / Fluent)

This is the simplest path if you're running from an executor. The bundled file (`VoidUI.lua`) is a single, self-contained script generated by `build_bundle.py` — it inlines every `Core/`, `Animations/`, `Themes/`, `Assets/`, `Managers/`, and `Widgets/` module and rewrites all the internal `require`s, so it behaves identically to the folder version. Host it anywhere reachable over HTTP (GitHub raw, a paste service, your own CDN) and load it in one line:

```lua
local VoidUI = loadstring(game:HttpGet("https://raw.githubusercontent.com/outhackernuls090-hash/VoidUI/refs/heads/main/VoidUI.lua"))()

local UI = VoidUI.new({ Name = "My UI", Startup = true })
UI:SetTheme("Midnight")

local Window = UI:CreateWindow({ Title = "VoidUI", Icon = "Home" })
local Tab = Window:CreateTab({ Title = "Combat", Icon = "Sword" })
Tab:CreateToggle({ Title = "Aimbot", Default = false, Callback = function(v) print(v) end })

UI:RunStartupSequence()
```

The loader is environment-agnostic: it works in Synapse X, Script-Ware, KRNL, Fluxus, and any executor that exposes `game`/`HttpGet`. If you need to fetch from a different host, just swap the URL — the bundle itself contains no environment-specific calls.

> **Regenerating the bundle:** after editing any module, run `python build_bundle.py` (or `py build_bundle.py`). It walks `VoidUI/`, rewrites the `require`s, and writes a fresh `VoidUI.lua`.

### Option B — Local module (recommended for development)

Keep the readable multi-file tree and `require` it directly:

```lua
local VoidUI = require(path.to.VoidUI)
```

Both options expose the exact same API.

## Quick Start

```lua
local VoidUI = require(script.Parent.VoidUI) -- or loadstring(...)()

local UI = VoidUI.new({
	Name = "My UI",
	Startup = true,          -- play the cinematic intro
	Theme = "Midnight",       -- optional; defaults to "Default"
	Accent = Color3.fromRGB(120, 180, 255),
})

local Window = UI:CreateWindow({
	Title = "VoidUI",
	Icon = "Home",
	Size = UDim2.fromOffset(560, 420),
	Position = UDim2.fromScale(0.5, 0.5),
})

local Combat = Window:CreateTab({ Title = "Combat", Icon = "Sword" })

Combat:CreateSection({ Title = "Aimbot" })

Combat:CreateToggle({
	Title = "Enable Aimbot",
	Default = false,
	Callback = function(Value) print("Aimbot:", Value) end,
})

Combat:CreateSlider({
	Title = "FOV",
	Min = 10, Max = 120, Default = 60,
	Callback = function(Value) print("FOV:", Value) end,
})

Combat:CreateDropdown({
	Title = "Target Priority",
	Options = { "Closest", "Lowest Health", "Crosshair" },
	Default = "Closest",
	Callback = function(Value) print("Priority:", Value) end,
})

UI:Notify({ Title = "Loaded", Description = "Welcome to the future.", Type = "Success" })
UI:RunStartupSequence()
```

See `Examples/Example.lua` for a fuller tour.

## The Public API

Everything below is callable on the object returned by `VoidUI.new(...)`.

### Library-level

| Function | What it does |
| --- | --- |
| `VoidUI.new(Options)` | Creates the root UI instance. `Options` accepts `Name`, `Startup`, `Theme`, `Accent`, and `ConfigName`. |
| `:CreateWindow(Options)` | Opens a new window and returns it. See the Window API below. |
| `:SetTheme(Name)` | Switches the active theme ("Default", "Midnight", "Nebula", "Crimson", "Emerald") and re-skins every open window. |
| `:SetAccent(Color)` | Recolors the entire UI with a new `Color3` accent, live. |
| `:Notify(Options)` | Pushes a toast notification. `Options`: `Title`, `Description`, `Type` (`Success`/`Warning`/`Danger`/`Info`), `Duration`. |
| `:RegisterPlugin(Plugin)` | Registers a plugin table (with `Name`, `Init`, optional `Unload`). |
| `:LoadPlugin(Name, Context)` | Loads a previously registered plugin by name. |
| `:RunStartupSequence(Callback)` | Plays the cinematic splash, then fires `Callback` (and the `"Ready"` event) when done. Safe to call once. |
| `:CreateCommandPalette()` | Returns a command-palette overlay you can populate with actions and open with a hotkey. |
| `:CreateContextMenu(Items)` | Returns a floating right-click style menu. `Items` is a list of `{ Label, Icon, Callback }` or `{ Separator = true }`. |
| `:CreateTooltip(Text)` | Returns a hover-following tooltip you can `:Attach(Instance)` to any element. |
| `:CreateModal(Options)` | Returns a centered dialog with a dimmed backdrop. `Options`: `Title`, `Description`, `Buttons` (list of `{ Label, Variant, Callback }`). |
| `:CreateBreadcrumbs(Items)` | Returns a horizontal path trail (`{ Label, Icon }` entries) with separators. |
| `:GetWindow(Title)` | Finds and returns an open window by its title. |
| `:SaveConfig()` / `:LoadConfig()` | Manually flush or restore the config managed by `ConfigManager`. |
| `:GetStats()` | Returns a table with window count, theme name, accent, asset-cache stats, plugin count, and an FPS estimate. |
| `:SubscribeReady(Callback)` | Runs `Callback` once the startup sequence finishes. |
| `:SubscribeThemeChanged(Callback)` / `:SubscribeAccentChanged(Callback)` | Fires whenever the theme or accent changes. |
| `:Destroy()` | Tears down every window, manager, and connection cleanly. |

### Window API

Returned by `:CreateWindow(...)`.

| Function | What it does |
| --- | --- |
| `:CreateTab(Options)` | Adds a sidebar tab (`Title`, `Icon`) and returns it. The first tab is selected automatically. |
| `:SelectTab(Tab)` | Switches focus to a given tab. |
| `:GetActiveTab()` | Returns the currently selected tab. |
| `:SetTitle(Title)` / `:SetSubtitle(Text)` / `:SetIcon(Name)` | Update the header text/icon live. |
| `:SetPosition(UDim2)` / `:SetSize(UDim2)` | Move or resize the window (animated). |
| `:SetZIndex(n)` | Change the window's stacking order. |
| `:Minimize()` / `:Restore()` | Collapse to the header or bring it back. |
| `:ToggleMaximize()` | Snap to (almost) fullscreen and back. |
| `:Unfold()` | Plays a spring "grow from nothing" reveal. |
| `:Close()` | Animates out and destroys the window, unregistering it from the manager. |
| `:IsClosed()` | Returns whether the window has been closed. |
| `:SubscribeChanged` / `:SubscribeTabChanged` / `:SubscribeClosed` | Event hooks for state changes. |

### Tab API (the widget factory)

Returned by `:CreateTab(...)`. Every `Create*` returns the widget instance so you can keep a reference and call its methods (`:SetValue`, `:SetVisible`, `:Destroy`, etc.).

**Layout & text**
- `:CreateSection({ Title })` — a collapsible titled group.
- `:CreateDivider()` — a thin separator line.
- `:CreateLabel({ Text })` — a static text line.
- `:CreateParagraph({ Title, Content })` — a heading plus body text block.

**Inputs**
- `:CreateButton({ Title, Variant, Callback })` — `Primary` / `Danger` / `Ghost` / `Icon`.
- `:CreateToggle({ Title, Default, Callback })` — on/off switch.
- `:CreateSlider({ Title, Min, Max, Default, Callback })` — numeric dragger.
- `:CreateTextbox({ Title, Default, Placeholder, Numeric, Password, MultiLine, Callback })` — text entry.
- `:CreateDropdown({ Title, Options, Default, MultiSelect, Callback })` — select-one or select-many.
- `:CreateSearchDropdown({ ... })` — a dropdown with a live filter box.
- `:CreateKeybind({ Title, Default, Callback })` — captures a key combination.
- `:CreateColorPicker({ Title, Default, Callback })` — saturation/value + hue picker.
- `:CreateGradientPicker({ Title, Default, Callback })` — multi-stop gradient editor.

**Data & feedback**
- `:CreateNotification({ Title, Description, Type })` — an inline alert card.
- `:CreateProgress({ Title, Value })` — a horizontal progress bar.
- `:CreateProgressRing({ Title, Value })` — a circular progress indicator.
- `:CreateAccordion({ Title, Items })` — a stack of expandable rows.
- `:CreateTreeView({ Title, Nodes })` — a collapsible node tree.
- `:CreateTable({ Columns, Rows, Sortable, Striped })` — a sortable data grid.

**Advanced / developer**
- `:CreateConsole({ Height })` — a log console with levels and a clear button.
- `:CreateTerminal({ Height })` — a command-line style box.
- `:CreateMarkdown({ Source })` — renders basic markdown.
- `:CreateCodeEditor({ Language, Height })` — line-numbered editor with a Run button.
- `:CreateGraph({ Points, Height })` — a line graph.
- `:CreateChart({ Kind, Data })` — `Bar` or `Pie` chart.
- `:CreateTimeline({ Events, Duration })` — a playhead with markers.
- `:CreateImage({ Url })` / `:CreateVideo({ Url })` / `:CreateAudio({ Url })` — media widgets.
- `:CreateFileExplorer({ Root })` — a browsable file/folder tree.
- `:CreateNestedTabs({ Tabs })` — tabs inside tabs.

Plus `:SetTitle(Title)` / `:SetIcon(Name)` / `:Destroy()` on the tab itself.

### Overlay widgets (from `:CreateCommandPalette` etc.)

- **CommandPalette** — `:Register(Action)`, `:RegisterMany(list)`, `:Open()`, `:Close()`, `:Toggle()`, `:SetHotkey(KeyCode)`.
- **ContextMenu** — `:Show(Position)`, `:Close()`, `:SetItems(Items)`.
- **Tooltip** — `:Attach(Instance)`, `:Show()`, `:Hide()`, `:SetText(Text)`.
- **Modal** — `:Open()`, `:Close()`.
- **Breadcrumbs** — `:SetItems(list)`, `:Push(item)`, `:Pop()`, and an `OnNavigate` callback.

## Theming

```lua
UI:SetTheme("Nebula")
UI:SetAccent(Color3.fromRGB(180, 120, 255))
```

Each theme exposes `Colors`, `Gradients`, `Typography`, `Layout`, `Effects`, and `Animation` tables that the whole framework reads from, so a theme swap is instant and consistent.

## Startup Sequence

Call `UI:RunStartupSequence()` (or pass `Startup = true` to `VoidUI.new`) to play the intro: a graphite splash fades in, a glowing core pulses while particles orbit it, the logo and title reveal, and a status line walks through boot steps before everything dissolves into your interface.

## Architecture

```
VoidUI/
├── VoidUI.lua            # single-file bundle for loadstring (generated)
├── build_bundle.py       # regenerates VoidUI.lua from the tree below
├── init.lua              # public API root
├── Core/                 # Utilities, Events, Scheduler, State, Cleanup,
│                         # Animation, Renderer, Window, Application
├── Animations/           # Spring, Tween, Physics, Blur, Glow, Particles
├── Themes/               # theme tables + ThemeManager
├── Assets/               # Icons, Fonts, Images
├── Managers/             # Config, Plugin, Notification, Input, Window,
│                         # Asset, Layout
└── Widgets/              # Base + 33 widget modules + init aggregator
Examples/
└── Example.lua           # runnable demonstration
```
